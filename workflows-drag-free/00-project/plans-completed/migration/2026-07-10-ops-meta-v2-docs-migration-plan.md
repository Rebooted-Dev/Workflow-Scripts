## 1. Proposal

After retiring `00-project/Drag-Free-v2/` and filing the completed design package under `workflows-drag-free/00-project/changelog/plans/`, an audit of live **`Workflow-Scripts/00-project/`** found additional **v2 / Drag-Free package evidence** still sitting in ops meta.

**Problem:** Agents and humans scanning live `00-project/` still find migration research, redirect troubleshooting, and repair evidence that belong with the **package-scoped** meta (`workflows-drag-free/00-project/`), while the **ops changelog** and **archive tarball** correctly remain at live meta.

**Goal:** Move Tier A package evidence into `workflows-drag-free/00-project/`, leave Tier B ops history in place, rewrite Tier C stale path framing, and keep a clear two-meta model:

| Meta root | Role |
|-----------|------|
| `Workflow-Scripts/00-project/` | **Ops meta** — whole-repo history, product/ops changelog, restore archives, system plans |
| `Workflow-Scripts/workflows-drag-free/00-project/` | **Package meta** — WDF library design, migration evidence, package troubleshooting |

### Success criteria

1. Tier A research/troubleshooting/build-evidence live under `workflows-drag-free/00-project/` (or are deliberately pointer-only if a stub is kept).
2. Live `00-project/changelog/` remains intact as the ops history (no bulk move of changelog type entries).
3. Live `00-project/build/archive/*promotion-snapshot*` and salvage inventory stay at ops meta.
4. Stale docs no longer claim `00-project/Drag-Free-v2/` is retained or that consolidation meta lives under Update-AI-Tools `Drag-Free-v2/00-project/`.
5. Both troubleshooting indexes (if used) and WDF changelog/TODO reflect the migration.
6. No loss of content (moves, not deletes; optional short pointer stubs at old research paths).

### Non-goals

- Moving the entire live `00-project/changelog/` into WDF.
- Moving multi-model / adversarial / deep-review system plans (Tier B).
- Re-running directory rationalization or changing `workflows-drag-free` workflow content.
- Deleting the promotion tarball or nested consolidated logs under ops `build/archive/`.
- Force-pushing or rewriting git history.

### Review basis and relevance

This review checked the draft against the current `fix/v2.0a-separate-legacy-and-v2` worktree, the live source inventories, the two metadata roots, the repair utility, the path validators, and the current changelog/TODO conventions. The baseline checks passed before migration: `check-moved-targets.sh`, `check-active-markdown-links.sh`, `check-workflow-skills.sh`, `tools/wf validate`, `tools/wf build --check`, `tools/wf build skills --check`, and `git diff --check`.

The plan is **still needed and feasible**, but it is not yet deterministic enough to execute. The final plan must narrow the package-owned inventory, make pointer and optional-scope decisions explicit, correct path-routing assumptions, and add post-move validation. No external provider or delegated reviewer output was used; findings below are local, read-only evidence.

### P0 — No blocker identified

No S0/S1 release-stopping or data-loss blocker was found in the current draft. The migration remains documentation/meta work, provided the preflight, no-clobber, and rollback controls below are added to the final plan.

### P1 findings

#### 1. S2/P1 — The research move map includes a whole-repo review record

**Rationale:** `00-project/research/2026-07-06-workflow-scripts-consolidated-review-report.md` audits the entire consolidated snapshot, including the skill layer, generic validation warnings, delegation plumbing, and system-level TODOs. Its scope is not package-only. Moving it into WDF would blur the ops/package boundary the plan is intended to establish.

**Actionable fix:** Keep that report at live `00-project/research/`. Move only the migration problem statement and dual-tree comparison into WDF research. Keep a live-meta link to the report from the final plan if needed, but do not create a duplicate.

#### 2. S2/P1 — The optional troubleshooting set is not package-scoped and must be explicitly deferred

**Rationale:** The three optional build entries concern the generic `tools/wf`, rationalized planning targets, and shared engineering-quality contracts. The optional environment entry concerns external-provider approval. These are Workflow-Scripts ops history, not WDF package evidence. Leaving the choice open violates the finalisation acceptance criterion that no execution ambiguity remains.

**Actionable fix:** Keep all four optional entries in live `00-project/troubleshooting/`; migrate only the four required redirect/path-drift entries. Record this decision in the final plan and WDF TODO so it is not re-litigated during execution.

#### 3. S2/P1 — The moved repair utility has a broken repository-root assumption and stale generated wording

**Rationale:** `00-project/build/drag-free-v2-separation/remediate_moved_targets.py` currently derives `ROOT` from `parents[3]`, which is the repository root before the move but `workflows-drag-free/` after the move. It also regenerates README text claiming root `workflows/` is active and `00-project/Drag-Free-v2/` is retained. A literal move therefore leaves the utility non-runnable from its planned destination and allows a later rerun to reintroduce false guidance.

**Actionable fix:** Update the utility before/while moving it to discover the Git repository root explicitly, keep its output directory at `workflows-drag-free/00-project/build/drag-free-v2-separation/`, and update the generated summary text to name `workflows-drag-free/` as active and the ops tarball as the only restore source. Add a syntax check and a documented non-destructive smoke check.

#### 4. S2/P1 — Pointer handling and relative context paths are ambiguous or incorrect

**Rationale:** The draft makes old research pointers optional, and it permits either removing or retaining old files. Existing live changelog/research records still refer to the old live paths. Also, from `workflows-drag-free/00-project/plans/`, `../../00-project/research/...` resolves back into the WDF metadata root, not the sibling live `Workflow-Scripts/00-project/research/` root.

**Actionable fix:** Make short `# Moved` pointer stubs mandatory for every migrated research and required troubleshooting file; update both indexes to the new canonical paths. In the final plan, use repository-root-qualified paths such as `00-project/research/...` and `workflows-drag-free/00-project/research/...` instead of ambiguous relative paths.

#### 5. S2/P1 — Tier C rewrite scope needs an evidence-based file list

**Rationale:** The current live `00-project/docs/agents/repository-map.md` is stale, and `00-project/AGENTS.md` still describes the retired consolidation workspace. `00-project/README.md` already has the correct active-library and archive pointer, so rewriting it wholesale would add unnecessary churn. `00-project/plans-completed/README.md` is still Drag-Free-only. The root README and `SHARING_AND_SYNC.md` contain current WDF guidance but do not require an automatic rewrite from this audit.

**Actionable fix:** Make the repository map, live AGENTS file, and plans-completed README mandatory edits; treat live README, root README, and `SHARING_AND_SYNC.md` as targeted grep/spot-check items, changing them only if a current false claim is found.

### P2 findings

#### 6. S3/P2 — Changelog-index ownership is misstated

**Rationale:** The draft says `00-project/changelog/index.md` is generated by `tools/wf build`, but the current CLI build command only generates `catalog.json` and `ROUTER.md`; bookkeeping commands insert changelog/troubleshooting rows separately. The WDF changelog index is maintained by the metadata conventions, not by the build freshness gate.

**Actionable fix:** Replace the generated-index claim with the actual ownership rule: add newest rows directly according to the local changelog convention, then reopen and validate both indexes. Do not invoke `tools/wf log` for this WDF package-meta change without first fixing its metadata-root routing.

#### 7. S3/P2 — Verification does not cover the full migration boundary

**Rationale:** The existing validators prove active WDF redirect/link health, but `check-active-markdown-links.sh` intentionally scans only `workflows-drag-free/`. It will not detect broken links in live `00-project/` records or verify that moved evidence is tracked rather than ignored. The current root `.gitignore` ignores `workflows-drag-free/00-project/build/`.

**Actionable fix:** Add explicit post-move assertions for source/destination hashes, pointer targets, live-vs-WDF index rows, archive retention, Git tracking/ignore state, targeted stale-current-guidance searches, and the full validator suite. Add a narrow `.gitignore` exception for the WDF separation-evidence directory before relying on `git status` as proof.

#### 8. S3/P2 — The final plan needs an explicit rollback and dirty-worktree boundary

**Rationale:** The nested repository already has unrelated working-tree changes (`changelog/index.md`, `plans/TODO.md`, and the draft plan/changelog entry). A bulk `mv` or broad index rewrite could accidentally absorb those changes.

**Actionable fix:** Require a preflight `git status --short`, destination collision/hash checks, and path-scoped edits. State that rollback is to restore the pre-move paths from Git before index/doc rewrites; do not reset, clean, or alter unrelated parent-repo changes.

The consolidated final plan should:

1. Move two package-owned research files, four required package troubleshooting files, and the separation evidence tree.
2. Keep the consolidated review report and all four optional troubleshooting entries in live ops meta.
3. Make pointer stubs mandatory and use repository-root-qualified paths.
4. Update the moved utility’s root discovery and generated README wording.
5. Define exact Tier C edits, narrow `.gitignore` tracking, rollback controls, and the complete verification command set.
6. Remain an active, unexecuted plan until the migration is actually performed and verified; do not file it as completed during plan review.
