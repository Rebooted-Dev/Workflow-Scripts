# Documentation Sync Summary Template

## Purpose
Provide a concise, verifiable summary of a documentation sync effort.

## Required Sections
1. Project goal and scope.
2. Codebase areas reviewed.
3. Priority-ordered doc changes (P0, P1, P2, P3).
4. File maps for navigation (directory tree, plus key module maps as needed).
5. Key improvements with evidence.
6. Known gaps or follow-ups (also priority-tagged).

## Reporting Guidelines
- List changes in descending urgency/importance: P0 first, then P1, P2, P3.
- Include at least one file map (directory tree); add module maps only when they materially improve navigation.
- Use concrete counts only when measured (files updated, new files added).
- Avoid unverified percentages or performance claims.
- Prefer short bullet lists and cite paths for major changes.
- Explicitly call out any "nice-to-have" work that was deferred to avoid scope creep.
- Include how parallel agents were used, if applicable.

## Acceptance Criteria
- Summary is factual, reproducible, and scoped.
- All claims can be traced to specific files or changes.
- Priority ordering is explicit and consistent.
- No marketing language or inflated metrics.
