# Planning Workflows

Use these workflows to move from idea or problem statement to a reviewed, executable implementation plan.

| ID | Workflow | Use When |
|---|---|---|
| `research-and-plan` | `00-research-and-plan.md` | You need research and an initial implementation plan. |
| `plan-review` | `01-plan-review.md` | You already have a draft plan and need evidence-backed review. |
| `finalise-plan` | `02-finalise-plan.md` | You have review feedback and need a final implementation plan. |
| `plan-review-and-finalise` | `03-plan-review-and-finalise.md` | You want review and finalisation in one workflow. |
| `architecture-design` | `04-architecture-design.md` | T2/T3 work needs a design brief, ADRs, and boundary decisions before build. |

For T3 greenfield work, start with `../setup/08-greenfield-mvp.md`; it calls the planning workflows in sequence and requires a deployed walking skeleton before feature slices.
