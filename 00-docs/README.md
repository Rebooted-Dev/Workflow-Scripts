# 00-docs: Workflow Documentation and Review Archive

This directory stores documentation about the `Workflow-Scripts` system itself.

## Purpose

- Keep review and audit reports for workflow quality and consistency.
- Keep analysis artifacts (for example, workflow-to-skills mapping).
- Keep remediation plans created from review findings.
- Preserve historical review documents in `old-reviews/`.

## What This Directory Is Not

- This is **not** a runnable workflow category.
- Files here are reference/governance artifacts, not execution instructions.

## How to Use

- Read these files when you need context on why workflow structure changed.
- Use them during maintenance, refactoring, or governance discussions.
- For actionable workflows, use directories like `01-planning/`, `02-build-code/`, `03-debug/`, and `05-review-audit/`.
